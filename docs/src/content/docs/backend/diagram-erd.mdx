## Entitas Utama

### Users

Menyimpan data user dengan role `admin` atau `pic`.

**Fields penting:**

- `id` (Primary Key)
- `employee_id` (Unique)
- `username` (Unique)
- `role` (admin, pic)
- `telegram_id` (Optional)

### ItemMasters

Data induk barang dengan informasi umum.

**Fields penting:**

- `id` (Primary Key)
- `model_code` (Unique)
- `category`
- `procurement_year`
- `pic_id` (Foreign Key ke Users)

### ItemUnits

Unit fisik dari item master dengan tracking status dan kondisi.

**Fields penting:**

- `id` (Primary Key)
- `unit_code` (Unique)
- `status` (available, in_transit, borrowed, transferred, sold, demolished)
- `condition` (good, damaged, broken)
- `location_id` (Foreign Key)

### Locations

Tempat penyimpanan barang.

**Fields penting:**

- `id` (Primary Key)
- `location_code` (Unique)
- `building_name`
- `floor`

### Requests

Permintaan perpindahan, peminjaman, atau operasi barang.

**Fields penting:**

- `id` (Primary Key)
- `request_code` (Unique)
- `request_type` (borrow, transfer, sell, demolish)
- `status` (pending, in_transit, approved, rejected, completed, cancelled)

### OfficialReports

Laporan resmi yang dihasilkan dari request.

**Fields penting:**

- `id` (Primary Key)
- `report_number` (Unique)
- `report_type`
- `file_path`
- `is_approved`

### AuditLogs

Jejak aktivitas penting untuk audit trail.

**Fields penting:**

- `id` (Primary Key)
- `action` (CREATE, UPDATE, DELETE, APPROVE, REJECT, LOGIN, LOGOUT, dll)
- `entity_type` dan `entity_id` (untuk tracking operasi pada entitas spesifik)

### ItemLogHistory

Riwayat perpindahan unit barang antar lokasi.

**Fields penting:**

- `unit_id` (dari unit mana)
- `from_location_id` dan `to_location_id` (lokasi asal dan tujuan)
- `moved_by_id` (user yang melakukan)
- `moved_at` (timestamp)

### Notifications

Pemberitahuan ke user atas aktivitas sistem.

**Fields penting:**

- `type` (request, report, password, system)
- `user_id` (user penerima notifikasi)

### PasswordResets

Approval untuk perubahan password.

**Fields penting:**

- `status` (pending, approved, rejected)
- `user_id` (user yang minta reset)
- `admin_id` (admin yang approve/reject)

## Praktik Migration

Saat ada perubahan skema:

1. Update file `prisma/schema.prisma`.
2. Jalankan `prisma migrate dev --name <nama_perubahan>`.
3. File migration akan terbuat otomatis di folder `prisma/migrations/`.
